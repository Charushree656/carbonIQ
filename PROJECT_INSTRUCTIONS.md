# CarbonIQ Development Rules

- Never write business logic inside React components.
- Use TypeScript strictly.
- Every feature lives in its own folder.
- Prefer reusable components.
- Never hardcode API URLs.
- Keep OCR, AI, and Carbon Engine as separate services.
- Mobile-first responsive design.
- Use shadcn/ui components whenever possible.
- Explain architectural decisions in comments only when necessary.
- Every new feature must compile without TypeScript errors.
